Relasi

  • auth.users → profiles — 1:1, dibuat otomatis saat signup (trigger).
  • profiles → workspaces — 1:N (owner). Tiap user bisa membuat banyak workspace.
  • profiles ↔ workspaces — M:N lewat workspace_members (membawa role, division, job_title). Di sinilah anggota ditambahkan.
  • workspaces → boards → lists → cards — rantai 1:N.
  • cards.workspace_id / board_id — denormalisasi (disinkron trigger) supaya cek hak akses (RLS) tanpa banyak join.
  • cards.assignee_id / created_by → profiles — penanggung jawab & pembuat kartu.

Model peran & akses

Platform (global per akun): super_admin dashboard khusus — lihat & kelola semua workspace/board, atur peran platform. user user biasa — punya/ikut workspace.

Workspace (per anggota, berlaku ke semua board di dalamnya): admin kelola anggota + semua isi member edit board/list/kartu viewer hanya lihat

division & job_title hanya label informatif (mis. Creative / Editor) — tidak mengubah hak akses. Hanya admin workspace yang bisa mengisinya. Akses board diturunkan dari keanggotaan workspace.
